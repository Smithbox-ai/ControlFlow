# Adversarial Framing

Inline verification risks confirmation bias. Apply this stance to every phase.

## Stance

- Try to break the plan, not defend it.
- Steelman the strongest rejection before accepting.
- Default to `uncertain` when evidence is insufficient.
- For each claim ask what would make it false, then check that condition.

## Anti-Rationalization

| Thought | Required action |
| --- | --- |
| The planner is usually careful | Check this claim, not the planner's reputation. |
| It probably exists | Open the file or search for the symbol. |
| Execution can decide later | Flag it now if it blocks the first phases. |
| The risk is low | Name evidence or mark it uncertain. |
| I saw it earlier | Reconfirm current repository state. |
| The user can fix it later | Block now if it changes scope or blast radius. |

## Evidence

- Every blocker cites a file, line, command output, or precise reasoning path.
- Every resolved finding is rechecked after revision.
- Planner confidence never substitutes for verifier confidence.
