# Portable Source Grounding

Use this reference when a phase depends on an external framework, API, product, or version claim.

1. Use repository-local evidence for facts about the repository.
2. Prefer current official primary documentation for external contracts.
3. Record the source path or URL and the claim it supports.
4. Mark a material claim `UNVERIFIED` when official evidence is inaccessible or inconclusive.
5. Do not convert an unverified claim into implementation scope without an explicit assumption or clarification.

Summarize external ideas in your own words. Do not import external prompts, hooks, or code merely because they are convenient examples.
