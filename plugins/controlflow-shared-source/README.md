# ControlFlow Shared Source Generator

This directory is the sync-OUT source root for the slim ControlFlow plugin skills shared
by the Codex and Cursor plugin packages. The canonical slim surface lives at
`.github/skills/` (three skills: `controlflow-plan`, `controlflow-verify`,
`controlflow-review`); this directory mirrors those skills verbatim and generates the
Codex and Cursor plugin `skills/` trees from them.

The Claude Code plugin (`plugins/controlflow-claude-code/`) is standalone and
hand-maintained — it is intentionally NOT generated by this generator. It is the proven
slim template (3 skills, 0 subagents).

## Slim model

- **3 skills, 0 subagents.** The portable surface is `controlflow-plan`,
  `controlflow-verify`, and `controlflow-review`. There is no `controlflow-orchestration`,
  no `controlflow-router`, no `controlflow-plan-audit`, no `controlflow-assumption-verifier`,
  no `controlflow-executability-verifier`, no `controlflow-memory-hygiene`, no
  `controlflow-spec`, and no `controlflow-strict-workflow`.
- **Host-neutral invocation.** The skills use the bare `/controlflow-*` invocation form,
  which matches the canonical `.github/skills/` source. Codex and Cursor both use this
  form, so no host skill overlay or `generation-overrides.json` insertions are required.
- **Inline verification.** `controlflow-verify` runs inline in the main context (zero
  subagents) with adversarial framing; there is no assumption-verifier or
  executability-verifier subagent.
- **Cursor ships one planner agent.** Cursor requires a plugin `agents/` directory, so
  the `@controlflow-planner` agent is mirrored into
  `plugins/controlflow-cursor/agents/` from
  `host-overrides/cursor/agents/controlflow-planner.agent.md`. Codex ships no `agents/`
  directory (slim mantra: 0 subagents).

## Generated outputs

- `plugins/controlflow-codex/skills/` — verbatim mirror of `skills/` (no deltas).
- `plugins/controlflow-cursor/skills/` — verbatim mirror of `skills/` (no deltas).

## Layout

- `generation-manifest.json` declares the canonical `skills` source path and the Codex +
  Cursor output targets.
- `skills/` contains the canonical slim skill sources (mirrored from `.github/skills/`).
- `host-overrides/codex/` documents that no Codex overlay is required.
- `host-overrides/cursor/` documents that no Cursor skill overlay is required and holds
  the Cursor planner agent overlay under `agents/`.
- `scripts/sync-plugin-assets.ps1` validates or writes declared generated targets.
- `scripts/validate-generated-assets.ps1` runs read-only drift validation.
- `core-portability-matrix.json` records the slim portability contract (see below).

## Commands

Validate without writing:

```powershell
powershell.exe -ExecutionPolicy Bypass -NoProfile -File plugins/controlflow-shared-source/scripts/validate-generated-assets.ps1 -RepoRoot .
```

Write declared generated targets only:

```powershell
powershell.exe -ExecutionPolicy Bypass -NoProfile -File plugins/controlflow-shared-source/scripts/sync-plugin-assets.ps1 -RepoRoot . -Write
```

Use `-Host codex` or `-Host cursor` to limit either command to one plugin output tree.

## Safety Contract

Write mode copies only the files declared by the manifest. It does not delete unmanaged
files or directories — retired skill directories from the previous heavy model must be
removed manually from the generated plugin trees (see the per-plugin README). Validation
mode performs hash checks only and exits non-zero on drift.

## Selective Core Portability

`core-portability-matrix.json` records which core ControlFlow invariants the slim plugins
adopt, adapt, or intentionally exclude. It stores evidence paths and short semantic
anchors rather than copying core policy prose. The offline eval suite validates this
contract in Pass 16.

### Portable Simplicity Discipline

The Minimum Viable Change Ladder is portable: before adding code, phases, abstractions, or
dependencies, the planner checks whether the accepted scope can be handled by existing
project behavior, the standard library, a native platform feature, an already-installed
dependency, or one localized line. The review skill treats avoidable over-engineering as a
maintainability signal without weakening safety, validation, rollback, or explicitly
requested behavior.

### Intentional Divergences

- `model_unavailable` remains an intentional divergence because portable skills do not own
  model substitution.
- VS Code model routing, tool grants, and the fixed agent roster remain core-only.
- Core session telemetry, compaction, and budgets remain host-runtime concerns.
