# Portable Decision Challenge

Use one bounded challenge for a high-risk or non-trivial decision whose failure would materially change scope, architecture, recovery, or user-visible behavior.

1. State the proposed decision and evidence.
2. State the strongest plausible failure mode.
3. Compare one simpler or safer alternative.
4. Keep, revise, or reject the decision and record why.

Stop after one challenge unless new evidence materially changes the decision. Do not invoke external agents, cross-model CLIs, or hidden review loops.
