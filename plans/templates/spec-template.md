# Spec Capture Template

Use this template for non-trivial, vague, or `SMALL+` tasks before final plan decomposition.

## Spec: {Task Title}

**Artifact Type:** SpecCapture
**schema_version:** 1.0.0
**Trace ID:** optional UUID

### Objective

- One concrete outcome the task must achieve.

### Scope In

- Files, behaviors, contracts, or docs that may change.

### Scope Out

- Explicit non-goals and areas that must not change.

### Constraints

- Technical, safety, compatibility, dependency, or time constraints.

### Acceptance Criteria

- Observable, automated criteria for success.

### Validation Commands

- Exact automated commands expected to verify completion.

### Open Questions

- Questions that block stable planning, each with owner or resolution path.
