## Шаблон главы

Каждая глава следует этой структуре:
- **Зачем эта глава** — что вы поймёте после чтения.
- **Ключевые понятия** — определения терминов, введённых в главе.
- **Mermaid-диаграмма** — визуальная модель описываемого процесса или структуры.
- **Подробное описание** — объяснение с примерами и ссылками на код.
- **Типичные заблуждения** — что чаще всего понимают неправильно.
- **Упражнения** — практические задания (🟢 новичок / 🟡 средний / 🔴 продвинутый).
- **Контрольные вопросы** — самопроверка.
- **См. также** — ссылки на связанные главы и файлы.

## Канонические источники

Всё содержимое глав произведено из:
- `.github/agents/controlflow-planner.agent.md` — единственный поставляемый агент; authoritative для поведения planner'а.
- `.github/copilot-instructions.md` — общий routing stub (таблица тиров, classification сбоев).
- `.github/skills/controlflow-plan/`, `.github/skills/controlflow-verify/`, `.github/skills/controlflow-review/` — три workflow skill'а.
- `governance/runtime-policy.json` — authoritative для tier-gated verify depth, semantic-risk policy и verdict routing.
- `governance/project-context-registry.json` — authoritative для концептуальной taxonomy ролей (исполнители + verify).
- `schemas/*.json` — authoritative для форм контрактов (anchor формата плана — `schemas/planner.plan.schema.json`).
- `docs/agent-engineering/NATIVE-DELEGATION-BOUNDARY.md` — authoritative для границы делегирования native-vs-ControlFlow.
- `plans/project-context.md` — authoritative человекочитаемое зеркало taxonomy ролей, тиров и конвенций.

Если пособие конфликтует с каноническим источником, канонический источник побеждает.

## Соглашения по тексту

- **`monospace`** — пути файлов, имена полей, enum-значения, команды.
- _Курсив_ — акцент на ключевой момент при первом упоминании.
- → — «см. также»-ссылка внутри или между главами.
- Технические термины (имена агентов, пути файлов, имена полей) — на английском throughout.
